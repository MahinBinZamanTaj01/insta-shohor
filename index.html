Questions & Answer Part

JavaScript is a client-side scripting, single-threaded, programming language and one of the most efficient. The term .client-side scripting language means that it runs at the client-side machine inside the web-browsers, but we have to always keep in mind that client's web-browser should support the JavaScript or it must be JavaScript enabled. In recent times, most of the modern web browsers support JavaScript and have their JavaScript engines like- Google Chrome has its own JavaScript engine called "V8", Edge has "Chakra", Safari has "JavaScript Core" and so on. We use JavaScript mainly for our web development jobs. Besides that we use JavaScript to validate data given by the user in the form fields.

In JavaScript, an event loop is method that pulls contents out of the queue and places it onto the function execution stack whenever the function becomes null. Event loop is like an illusion that makes JavaScript look like multi-threaded, even though it's a single-threaded programming language. When the function stack becomes empty, the function gets loaded onto the stack by the help of event loop. The Event Loop has one simple job and that is to monitor the call stack and the callback Queue. Event-loop has different timer functions (like- setTimeout() and setInterval()) that helps us to set the time to load data.

Local Strorage and Session Storage are very much similar. Local Strorage and Session Storage- are properties that allow JavaScript sites and apps to save key-value pairs in a web browser. The main difference between these is the expiration time. Local storage has no expiration time whereas Session storage has a expiration time which is one session. If we close a tab, the data of that tab which is in the local storage is affected by this. On the other hand, we loose data from the session storage if the tab is closed.
